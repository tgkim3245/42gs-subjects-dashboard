# UI/UX 화면 설계 및 컴포넌트 명세서

> **문서 버전:** v0.2 (결정 사항 반영)
> **최종 수정일:** 2026-07-01
> **작성자:** -

---

## 1. 개요

이 문서는 42-subject-dashboard 크롬 익스텐션의 UI 구성 요소, 색상 시스템, 인터랙션 규칙을 정의합니다.

**디자인 원칙:**
- **가독성 최우선**: 빽빽한 행렬 데이터를 마우스 오버 하이라이트로 가독성 확보
- **진행 상태 가시화**: API 동기화 진행률을 항상 노출
- **다크 테마 기본**: 눈의 피로를 줄이고 40 Intra 느낌과 통일감 유지

## 2. 전체 화면 레이아웃

### 2-1. 모달(Modal) 창 크기 및 렌더링 방식

| 항목 | 값 / 렌더링 방식 |
| :--- | :--- |
| **너비** | `100vw` (전체 화면 너비) |
| **높이** | `100vh` (전체 화면 높이) |
| **렌더링 방식** | Content Script(`injector.js`)가 인트라넷 페이지에 불투명한 Full-screen `iframe` 모달을 주입 |
| **오버레이 배경** | `rgba(15, 17, 23, 0.98)` (98% 불투명 다크 테마 배경 - 기존 인트라넷을 덮음) |
| **스타일 격리** | `iframe`을 사용하여 인트라넷 원본 사이트의 CSS 오염 및 스타일 충돌을 완벽히 차단 |

### 2-2. 레이아웃 구조

```
┌──────────────────────────────────────────────────────────────┐
│  [🔷 42 Subject Dashboard]  [설정 ⚙]  [새로고침 🔄]  [닫기 ❌] │  ← Header (높이 48px)
│  (최종 동기화 완료: 2026-07-01 21:20:00)                      │  ← Header Subtitle (글로벌 갱신 정보)
├──────────────────────────────────────────────────────────────┤
│  [전체 보기] [⭐ 관심 카뎃]                    [🔍 검색창...]  │  ← Tab & Search Bar (높이 40px)
├──────────────────────────────────────────────────────────────┤
│  정렬 기준: [레벨 ▼] [블랙홀 ▼] [과제명 클릭 정렬]            │  ← Sort Controls (높이 36px)
├──────────────────────────────────────────────────────────────┤
│                                                              │
│              📊 대시보드 테이블                              │  ← Dashboard Table (스크롤 영역)
│                                                              │
├──────────────────────────────────────────────────────────────┤
│  [████████░░░░░░░ 데이터 동기화 중... 12/50 완료]              │  ← Progress Bar (높이 32px)
└──────────────────────────────────────────────────────────────┘
```

* **닫기 [❌] 버튼 기능:**
  * 클릭 시 `parent.postMessage({ type: "CLOSE_DASHBOARD" }, "*")` 메시지를 송신합니다.
  * Content Script(`injector.js`)가 메시지를 수신하여 주입된 `iframe` 요소를 DOM에서 삭제하거나 `display: none` 처리하여 원래의 인트라넷 화면으로 돌아갑니다.

### 2-3. 사용자 체감형 데이터 갱신 정보 UI 명세

사용자가 데이터의 신선도와 실시간 업데이트 상태를 투명하게 인지할 수 있도록 다음과 같은 피드백 UI를 제공합니다.

#### ① 글로벌 최종 동기화 시간 (Global Last Sync Timestamp)
* **표시 위치:** 대시보드 좌측 상단 헤더 타이틀 아래 (Subtitle 영역)
* **표시 형식:** `최종 동기화 완료: YYYY-MM-DD HH:MM:SS` (또는 `방금 전`, `5분 전` 등 상대적 시간 표시)
* **동작:** 전체 백그라운드 동기화 또는 개별 카뎃 정보 갱신 시점에 실시간으로 타임스탬프가 최신화됩니다.

#### ② 개별 카뎃 정보 갱신 상태 배지 (Row Status Indicator)
* **표시 위치:** 테이블 행의 카뎃 ID(Login) 좌측 영역
* **상태별 표현:**
  * **로딩 중 (Syncing):** 회전하는 스피너 아이콘(Spinner)을 노출하여, 현재 해당 카뎃의 데이터가 42 API를 통해 갱신되고 있음을 시각화합니다.
  * **최신 상태 (Fresh):** 저장된 데이터가 TTL(2시간) 이내인 경우 별도의 배지를 표시하지 않거나, 마우스 오버 시 `최종 갱신: N분 전` 툴팁을 제공합니다.
  * **완료 직후 피드백 (Animation):** 백그라운드에서 수집이 완료되어 값이 변동되는 순간, 해당 행(Row) 전체에 1.5초간 연한 파란색 페이드인-아웃(Fade-in/out) 배경 애니메이션 효과를 주어 데이터가 성공적으로 업데이트되었음을 사용자에게 직관적으로 전달합니다.

/* 현재 마우스 오버 셀의 열 전체 하이라이트 (JS 보조 필요) */

- 마우스가 특정 셀에 들어오면 해당 열 index를 JS로 추적

## 5. 검색, 필터 및 정렬 시스템

본 익스텐션의 핵심 기능 중 하나는 방대한 카뎃 목록을 직관적으로 정렬하고 필터링하는 것입니다.

### 5-1. 검색 (Search)
* **위치:** 대시보드 탭바 우측 영역
* **작성 방식:** 입력即시 필터링 적용 (200ms 디바운스 적용하여 성능 최적화)
* **검색 대상:** 카뎃의 Login ID
* **조건:** 대소문자 구분 없이 부분 일치만 해도 노출

---

### 5-2. 필터링 (Filtering)
사용자는 여러 필터를 조합하여 타겟 카뎃 그룹을 압축할 수 있습니다.

#### ① 기수 필터 (Cohort / Generation Filter)
* **기본 원리:** `users_index`의 `begin_at` (본과정 시작 날짜) 값을 기준으로 그룹화합니다.
* **매핑 방식:**
  * **42 Gyeongsan (경산 캠퍼스):** 알려진 공식 기수 시작일 기준으로 기수 이름을 자동 변환하여 제공합니다.
    * `2024-02-26` → **1기**
    * `2024-10-01` → **2기**
    * `2025-04-01` → **3기 A**
    * `2025-10-01` → **3기 B**
    * `2025-12-15` → **3기 C**
    * `2026-04-01` → **4기 A**
    * (이후 기수는 운영 일정에 맞춰 추가)
  * **기타 알 수 없는 날짜:** 본과정 시작일의 **YYYY-MM-DD** 단위로 자동 그룹화하여 필터 목록을 동적 생성합니다.
* **UI 형태:** 필터바에 수평으로 나열된 **Pill Checkbox** 방식 (다중 선택 가능). 드롭다운 미사용.

#### ② 과제 상태 필터 (Project Status Filter)
* 특정 과제를 선택한 뒤 해당 과제의 진행 상태별로 필터링할 수 있습니다.
* **필터 대상 과제:** Circle 00 ~ Circle 06 과제 중 택 1
* **필터 상태 조건:**
  * **전체 보기** (기본값)
  * **통과자만 보기** (`validated` 이고 점수가 입력됨)
  * **진행중만 보기** (`in_progress` 상태)
  * **실패자만 보기** (`finished` 이나 `validated`가 false)
  * **미시작자만 보기** (해당 과제 기록 자체가 없음)

#### ③ 블랙홀 필터 (Blackhole Filter)
* 블랙홀 제적 여부를 기준으로 스위치 토글로 제어합니다.
* **토글 의미:**
  * **블랙홀 포함(ON):** 블랙홀 기간이 만료되어 제적 처리된 카뎃도 테이블에 표시 (기본값)
  * **블랙홀 제외(OFF):** 제적된 카뎃 목록에서 숨김
* **블랙홀 남은 날짜 정보는** 아이디 위에 마우스 오버 시 툴틱으로 확인 가능 (필터 단계구분 불필요)

#### ④ 레벨 구간 필터 (Level Range Filter)
* 서클 진척도와 직결되는 본과정 레벨별로 카뎃을 분류합니다.
* **필터 옵션:**
  * **전체 레벨** (기본값)
  * **초기 서클 (Lv 0.00 ~ 1.99)** (Circle 00 ~ 01 위주)
  * **중기 서클 (Lv 2.00 ~ 3.99)** (Circle 02 ~ 03 위주)
  * **아우터 서클 진입 (Lv 4.00 ~ 5.99)** (Circle 04 ~ 05 위주)
  * **아우터 서클 마스터 (Lv 6.00 이상)** (Circle 06 및 졸업 준비 단계)

#### ⑤ 현재 진행 서클 필터 (Current Circle Filter)
* 카뎃이 현재 수행 중인(아직 통과하지 못한 가장 낮은) 서클을 기준으로 카뎃을 필터링합니다. 이는 레벨 필터보다 엄격한 과제 완료 여부 기준의 필터입니다.
* **UI 형태:** 테이블 헤더의 Circle 그룹 헤더를 **클릭 토글**하여 필터 적용. 1수 클릭: 해당 서클 필터 활성화, 2수 클릭: 해제.
  * **Circle 0**: Libft 미통과자
  * **Circle 1**: Circle 0 통과 완료, Circle 1 과제(ft_printf, get_next_line) 중 하나라도 미통과 상태
  * **Circle 2**: Circle 1 통과 완료, Circle 2 과제(push_swap, minitalk) 중 하나라도 미통과 상태
  * **Circle 3**: Circle 2 통과 완료, Circle 3 과제(philosophers, minishell) 중 하나라도 미통과 상태
  * **Circle 4**: Circle 3 통과 완료, Circle 4 과제(cub3d, miniRT) 중 선택과제 1개 이상 미통과 상태 (선택지 중 하나만 통과 시 통과로 간주)
  * **Circle 5**: Circle 4 통과 완료, Circle 5 과제(ft_irc, Inception) 중 하나라도 미통과 상태
  * **Circle 6**: Circle 5 통과 완료, Circle 6 과제(ft_transcendence) 미통과 상태

### 5-3. 정렬 규칙 (Sorting)
테이블 헤더 영역을 클릭하면 직관적으로 데이터가 정렬됩니다.

| 정렬 대상 (헤더 클릭) | 1차 클릭 정렬 방식 | 2차 클릭 (토글) | 정렬 시 우선순위 예외 처리 |
| :--- | :--- | :--- | :--- |
| **Login ID** | 알파벳 오름차순 (A-Z) | 알파벳 내림차순 (Z-A) | 없음 |
| **Level** | 레벨 내림차순 (높은 순) | 레벨 오름차순 (낮은 순) | 없음 |
| **Blackhole** | 남은 날짜 오름차순 (임박 순) | 남은 날짜 내림차순 (여유 순) | 블랙홀 유예/없음 카뎃은 오름차순 시 맨 하단에 위치 |
| **Start Date** | 본과정 시작일 내림차순 (최신 기수 우선) | 본과정 시작일 오름차순 (과거 기수 우선) | 날짜 정보 누락 시 맨 하단 |
| **과제 셀 헤더 (예: Libft)** | 과제 점수 내림차순 (고득점 순) | 과제 점수 오름차순 (저득점 순) | 1. 통과(점수 높은 순)<br>2. 진행중<br>3. 미시작/실패 (정렬 순서 뒤로 배치) |

* **기본 정렬값:** 레벨 내림차순 (`Level` ▼)

## 6. 탭 컴포넌트

| 탭명 | 내용 |
| :--- | :--- |
| 전체 보기 | 캠퍼스 전체 카뎃 대시보드 (전체 스크롤, 400명 미만이므로 Pagination 불필요) |
| ⭐ 관심 카뎃 | 대시보드 테이블 내 별표(☆/★) 클릭으로 등록한 카뎃만 필터링하여 표시 |

**페이지당 표시 인원:**
- 전체 스크롤 (Pagination 없음)
- 인원 400명 이상 증가 시 가상 스크롤(Virtual Scroll) 도입 검토

**성능 최적화 방안:**
- `contain: layout style` CSS 적용으로 리플로우 범위 제한
- 필터 변경 시 16ms debounce 후 적용
- `DocumentFragment` 일괄 DOM 삽입 (실데이터 로드 시)

## 8. Intra 프로필 페이지 주입 버튼

Content Script(`injector.js`)가 42 Intra 프로필 페이지에 주입하는 버튼입니다.

**위치:** 프로필 페이지 상단, 기존 "Send message" 버튼 우측에 삽입

```html
<button id="ext-bookmark-btn" class="ext-bookmark-btn">
  ⭐ 관심 카뎃 추가
</button>
```

**동작:**
1. 클릭 시 현재 프로필 유저의 `login`을 `chrome.storage.local`에 북마크 목록에 추가
2. 이미 추가된 경우 버튼 텍스트를 `✅ 관심 카뎃 등록됨`으로 변경
3. 다시 클릭 시 북마크 해제
